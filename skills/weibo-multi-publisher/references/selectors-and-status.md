# 微博当前已验证状态

## 已实测打通
- 扫码登录 → 保存 Playwright storage state
- 使用登录态进入首页
- 纯文案发微博成功
- 单视频 + 文案微博成功

## 当前页面要点
微博首页发文区当前不是 `contenteditable`，而是：

- 输入框：`textarea[placeholder="有什么新鲜事想分享给大家？"]`
- 发送按钮：`button:has-text("发送")`

## 成功判定
发文后回读首页正文，如果能看到刚发送的文案，则视为成功。

## 风险
- 微博前端 class 经常变，不要依赖动态 class 名
- 优先用 placeholder / 按钮文本做选择器
- 多账号同文案同时间发送可能触发限制

## 建议 rollout
1. 先单账号验证
2. 再扩展到三账号轮发
3. 图文和视频分开接，不要跟纯文案混在第一版里
